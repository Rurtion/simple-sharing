# 微信读书导出工具

通过Hook Web页面中的Canvas函数，获取绘制到Canvas中的文本及样式等信息，转换成markdown格式，保存到本地文件，然后再转换成最终的epub或pdf格式，而mobi格式则是使用kindlegen工具从epub格式转换来的。原始内容来源于: https://github.com/drunkdream/weread-exporter。

## 环境要求

### 系统依赖

PDF 导出依赖 `weasyprint`，需要系统安装 Cairo / Pango：

- **macOS**:
  ```bash
  brew install cairo pango
  ```
- **Ubuntu/Debian**:
  ```bash
  sudo apt-get install libcairo2-dev pkg-config python3-dev
  ```
- **Windows**: 已内置所需 DLL，无需额外安装

### Python 版本

Python >= 3.7

## 安装

### 方式一：pip（推荐）

```bash
pip install -r requirements.txt
```

### 方式二：pip 可编辑模式安装

```bash
pip install -e .
```

### 方式三：uv

```bash
uv sync
uv run python -m weread_exporter ...
```

## 获取书籍ID

在页面 [https://weread.qq.com/](https://weread.qq.com/) 搜索目标书籍，进入书籍介绍页，URL格式为：

```
https://weread.qq.com/web/bookDetail/08232ac0720befa90825d88
```

其中 `08232ac0720befa90825d88` 就是书籍ID。

## 使用

### 基本用法

```bash
python -m weread_exporter -b <book_id> -o epub -o pdf
```

### 参数说明

| 参数 | 说明 |
|------|------|
| `-b, --book-id` | 书籍ID（必填） |
| `-o, --output-format` | 输出格式，可选值：`md`, `epub`, `pdf`, `mobi`, `txt`（可多次指定，如 `-o epub -o pdf`），默认 `epub` |
| `--force-login` | 强制重新登录（默认关闭） |
| `--headless` | 无头模式启动 Chrome（默认关闭，即显示浏览器窗口） |
| `--use-default-profile` | 使用 Chrome 默认用户配置（可复用已登录的微信读书会话，默认关闭） |
| `--proxy-server` | HTTP 代理服务器地址，如 `http://127.0.0.1:8888` |
| `--css-file` | 自定义 CSS 样式文件路径 |
| `--load-timeout` | 章节页面加载超时时间（秒，默认 60） |
| `--load-interval` | 章节加载间隔时间（秒，默认 30） |
| `--mock-user-agent` | 使用模拟 User-Agent（默认关闭） |

### 输出文件

生成的文件在项目根目录的 `output/` 文件夹中：

- `epub` 格式适合手机端访问
- `pdf` 格式适合电脑端访问
- `mobi` 格式适合 Kindle 访问（仅 Linux 支持转换）
- `txt` 格式纯文本
- `md` 格式 Markdown 源文件（存储在 `cache/<book_id>/chapters/`）

### 登录说明

首次使用需要登录微信读书账号：

1. 不加 `--use-default-profile` 时，工具会启动 Chrome 并打开微信读书登录页
2. 扫描二维码完成登录后，会话 Cookie 会保存在 `cache/cookie.txt` 中
3. 后续再次运行会自动复用 Cookie，无需重复登录
4. 如果 Cookie 过期，可加 `--force-login` 参数强制重新登录

如果想复用你已在 Chrome 上登录的微信读书会话，添加 `--use-default-profile` 参数：

```bash
python -m weread_exporter -b <book_id> -o epub --use-default-profile
```

> **注意**：Chrome 136+ 版本可能不支持 `--use-default-profile`，会遇到 sandbox 相关问题。

### 完整示例

```bash
# 导出为 epub + pdf
python -m weread_exporter -b 08232ac0720befa90825d88 -o epub -o pdf

# 无头模式导出（不显示浏览器窗口）
python -m weread_exporter -b 08232ac0720befa90825d88 -o epub --headless

# 使用代理
python -m weread_exporter -b 08232ac0720befa90825d88 -o epub --proxy-server http://127.0.0.1:8888
```
